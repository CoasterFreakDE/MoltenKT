Interchange

fun Interchange(label: String, aliases: Set<String> = emptySet(), protectedAccess: Boolean = false, userRestriction: InterchangeUserRestriction = NOT_RESTRICTED, accessProtectionType: InterchangeAuthorizationType = JET, hiddenFromRecommendation: Boolean = false, completion: InterchangeStructure = de.jet.paper.structure.command.completion.emptyInterchangeStructure(), ignoreInputValidation: Boolean = false, preferredVendor: App? = null)
Content copied to clipboard

Parameters

label

is the interchange name, used as / and as the identifier, defined inside your plugin.yml

aliases

are the supported interchange-aliases, which can be used, to access the interchange

protectedAccess

defines, if the InterchangeExecutor requires to have the permission 'interchange.

userRestriction

defines, which types of InterchangeExecutor are allowed, to execute the interchange

accessProtectionType

defines the system, which is used, to check the InterchangeExecutors ability to execute the interchange

hiddenFromRecommendation

if enabled, this interchange will not be visible in the tab-recommendations of usable commands

completion

defines the completions, that the interchange will display, during a InterchangeExecutors input into console/chat

ignoreInputValidation

defines, if the input of the user should not be checked of correctness

preferredVendor

if not null, this overrides the automatic extrapolated vendor on registering
